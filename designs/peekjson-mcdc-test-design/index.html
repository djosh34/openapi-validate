peekjson MC/DC Test Design

Target: pkg/peekjson/peekjson.go. This is a test design only; it does not add executable tests.

Intent

peekjson.Decoder wraps encoding/json.Decoder so callers can inspect the next JSON token with Peek() and later consume the same bytes through the public decoder. The implementation has two readers: peekReader reads from the source and copies bytes into lookAheadBuffer, while publicReader should replay that buffer before reading fresh source bytes.

Current status: Peek() does not assign d.peekedToken on success, so repeated peeks advance instead of returning the cached token. publicReader.Read is still a TODO and currently returns 0, nil, which can stall consumers. The test plan below is therefore written as a design for the intended implementation and includes tests that will fail against the current code.

Coverage Standard

The current code has only simple boolean decisions, so MC/DC is equivalent to proving both outcomes of each predicate. For the intended publicReader.Read, the design treats each buffer/source predicate independently so future compound conditions still have explicit true/false pairs.

Branch coverage Every if decision has tests for true and false paths.
MC/DC When a decision has multiple conditions, each condition must independently change the decision outcome while other inputs are held stable.
Behavioral oracle Peeking must not change the token stream observed by Token, Decode, or More.
Reader contract Any read returning n > 0 must preserve those bytes, even when the same read also returns an error.

Decision Catalog

ID Location Decision Required outcomes Notes
D1 Peek, line 31 d.peekedToken != nil true: cached token returned. false: calls peekDec.Token(). Requires a successful peek to cache peekedToken. Current code misses this assignment.
D2 Peek, line 36 err != nil after peekDec.Token() true: error stored and returned. false: token returned and cached. Use valid input for false, empty/invalid/truncated JSON for true.
D3 peekReader.Read, line 50 err != nil from source reader true: returns source error. false: writes read bytes into look-ahead buffer. Edge case: source may return n > 0 and err != nil; bytes must still be buffered before propagating error.
D4 intended publicReader.Read lookAheadBuffer.Len() > 0 true: read from replay buffer. false: read directly from source. This is the core no-consume guarantee after Peek.
D5 intended publicReader.Read buffer satisfies requested read vs only partially fills it true: return only buffered bytes. false: combine buffered bytes with source bytes or perform a second source read. Exact implementation can return after buffer bytes or fill the remainder; tests should assert byte order and no loss, not over-specify chunking unless the code commits to it.
D6 intended publicReader.Read err != nil from direct source read true: propagate error according to io.Reader rules. false: return source bytes. Include n == 0 with error and n > 0 with error.
D7 intended publicReader.Read public read invalidates a cached peek before read: cache hit. after read: next peek must read the next public token. Not a branch if implemented unconditionally, but critical state coverage.

MC/DC Matrix

Decision False case True case Independence argument
D1 T01: first Peek() on {"a":1}. T02: second Peek() before any public read. Only cache state changes; input and decoder position are otherwise held stable.
D2 T01: valid JSON has a next token. T08: empty reader returns io.EOF; T09: invalid token returns syntax error. Only parser result changes from token to error.
D3 R01: source read returns n > 0, nil. R02: source read returns 0, io.ErrUnexpectedEOF. Same reader path and buffer state; source error value toggles the branch.
D3 edge R01: successful bytes are buffered. R03: source returns n > 0, io.EOF; bytes are still buffered and error is returned. Guards against the common bug where bytes are dropped when n and err arrive together.
D4 P01: no look-ahead, public decoder reads source. P02: after Peek(), public decoder reads replay buffer. Only lookAheadBuffer.Len() changes; source JSON and destination buffer size stay stable.
D5 P03: look-ahead buffer shorter than caller buffer. P04: look-ahead buffer at least as long as caller buffer. Replay buffer length independently changes whether source bytes are needed for the call.
D6 P01: direct source read succeeds. P05: direct source read returns 0, io.EOF; P06: direct source read returns n > 0, io.EOF. Same direct-source path; only source result toggles error handling.
D7 T02: second peek without public read is a cache hit. T03: peek after consuming a token is not a cache hit and observes the next token. Only the public read between peeks changes cache validity.

Edge Cases

Category Cases Why it matters
JSON grammar empty input, whitespace-only input, invalid delimiters, unterminated string, truncated object, trailing garbage after one value. Ensures peek errors match public decoder errors and buffered bytes do not mask syntax failures.
Token variety object delimiters, array delimiters, object keys, strings with escapes, booleans, null, integers, decimals, exponents, very large integers. Confirms Peek mirrors encoding/json.Decoder.Token while preserving json.Number.
Reader behavior one byte per read, large read-ahead by json.Decoder, zero-byte reads, n > 0 with error, EOF after replay buffer. Most bugs in this package will be byte-replay bugs, not JSON parsing bugs.
State sequencing peek-peek, peek-token-peek, peek-decode-peek, token-peek-token, peek-more-decode, decode until EOF then peek. Validates cache invalidation and confirms public methods consume the same stream a normal decoder would consume.
Contract boundaries nil reader, concurrent calls, and mutation of embedded decoder options. These should be documented as unsupported unless the package explicitly commits to supporting them. Do not over-test undefined behavior.

Pragmatic Test Structure

  • Use one same-package file such as pkg/peekjson/peekjson_test.go for reader internals and public API tests that need state inspection.
  • Add helper readers: chunkReader, scriptedReader, and countingReader. Keep them tiny and deterministic.
  • Compare public token streams against encoding/json.NewDecoder for broad integration cases, with the known difference that peeked numbers should be json.Number.
  • For reader tests, assert byte sequences and state transitions directly instead of relying on statement coverage alone.
  • Run tests with the race detector only if concurrency support is added later; the current type should be treated as not goroutine-safe.

Acceptance Checklist

Future Coverage Command

Go's built-in coverage is statement-oriented, so it should be treated as a sanity check, not proof of MC/DC. The decision matrix above is the proof obligation. After implementation, run: 

go test ./pkg/peekjson -covermode=count -coverprofile=coverage.out
go tool cover -func=coverage.out
